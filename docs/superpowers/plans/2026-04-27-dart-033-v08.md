# DART-033 — flog_dart v0.8 SSE architecture redesign

> REQUIRED SUB-SKILL: superpowers:subagent-driven-development or executing-plans.

**Goal:** Resolve DART-033's 7 architecture issues via a layered StreamTransformer design. Ship as **flog_dart v0.8.0** — a breaking release with an explicit migration path. Non-SSE APIs (FlogLogger, FlogDio non-sse, FlogWebSocket, FlogMockInterceptor, Flog.init) remain source-compatible.

**Starting state:** flog_dart 0.7.3 at HEAD (`77ff975` + this plan). Current SSE lives in `flog_sse_parser.dart` (417 lines, mixed concerns) and `flog_dio_sse.dart` (77 lines, thin convenience).

**Architectural target:**

```
bytes              ─┐
                    │ SseByteDecoder : StreamTransformer<List<int>, String>
utf-8 text chunks  ─┤   (zero-copy UTF-8 boundary handling; bounded buffer)
                    │
                    │ SseLineDecoder : StreamTransformer<String, SseEvent>
SseEvent stream    ─┤   (pure W3C parser, stateful transformer)
                    │
                    │ ──(optional)── FlogSseReporter : StreamTransformer<SseEvent, SseEvent>
                    │                   (telemetry tee; passthrough when flogDisabled)
                    │
consumer's stream  ─┘
```

Three independent transformers composed by `FlogDio.sse()`. Each testable in isolation, each swappable / disableable.

**Red line:** 0.7.x characterization tests (47 in `flog_sse_parser_test.dart`) must keep passing against a v0.7.x-compat shim layer. Any behavior divergence is a bug to fix, not a test to update.

### Task 1 — `SseByteDecoder` : `StreamTransformer<List<int>, String>`

Create `flog_dart/lib/src/sse/byte_decoder.dart`. Wrap the UTF-8 boundary handling + BOM-strip logic currently embedded in `_run()`:

```dart
class SseByteDecoder extends StreamTransformerBase<List<int>, String> {
  final int maxBufferBytes; // default 1 MiB; throw if exceeded
  // stateful: maintains a Uint8List byteBuffer across chunks
  // yields decoded strings when a boundary is safe
}
```

- Use `Uint8List` + view slicing (no `.sublist` allocations in the hot path).
- Hard cap: if buffer grows past `maxBufferBytes`, close the stream with a descriptive error. Default 1 MiB, configurable.
- BOM strip on first chunk.

+ 8 tests: empty stream, single-chunk ASCII, multi-chunk ASCII, UTF-8 split across chunks (2-byte char, 3-byte char, 4-byte emoji), BOM prefix, buffer overrun triggers error, byte-level backpressure preserved.

Commit: `feat(flog_dart/sse): SseByteDecoder transformer (DART-033 v0.8 step 1)`

### Task 4 — Rewrite `FlogSseParser` as thin composition

`flog_dart/lib/src/flog_sse_parser.dart` becomes a **compatibility shim** for the 0.7.x API:

```dart
class FlogSseParser {
  /// v0.7.x-compat: Stream<String> of data payloads only.
  /// Equivalent to: byteStream.transform(SseByteDecoder()).transform(SseLineDecoder())
  ///                          .transform(FlogSseReporter(url: url, method: method))
  ///                          .map((e) => e.data).where((d) => d != '[DONE]');
  static Stream<String> wrap(Stream<List<int>> byteStream, {required String url, String method = 'GET'});

  /// v0.7.x-compat: typed Stream<SseEvent>.
  static Stream<SseEvent> wrapTyped(Stream<List<int>> byteStream, {required String url, String method = 'GET'});
}
```

Behavior MUST match 0.7.x `flog_sse_parser_test.dart` byte-for-byte. If a test fails, fix the composition (not the test).

The internal `_SseLineParser` + `_run` are deleted (logic now lives in `SseLineDecoder`).

Commit: `refactor(flog_dart/sse): FlogSseParser becomes v0.7-compat shim (DART-033 v0.8 step 4)`

### Task 5 — `FlogDio.sse()` v0.8 API

In `flog_dart/lib/src/flog_dio_sse.dart`:

```dart
class SseResponse {
  final Stream<String> stream;       // v0.7 compat (data-only)
  final Stream<SseEvent> events;     // v0.8 new: typed event stream
  final Headers headers;
  final int? statusCode;
  final RequestOptions options;
  // cancel() unchanged
}
```

`FlogDio.sse()` populates BOTH `stream` and `events`. Internally, ONE decoded `Stream<SseEvent>` is built; `stream` is `events.map(...)`. Users on 0.7 keep using `.stream`; users upgrading can switch to `.events`.

Deprecation annotation on `SseResponse.stream`:
```dart
@Deprecated('Use .events for typed access; removed in v1.0')
final Stream<String> stream;
```

+ 3 integration tests: .stream matches legacy behavior; .events surfaces event:/id:/retry: fields; both streams consume the same underlying byte stream (single subscription).

Commit: `feat(flog_dart/dio): SseResponse.events typed access (DART-033 v0.8 step 5)`
