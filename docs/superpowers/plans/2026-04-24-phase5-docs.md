### Task 1 — `docs/ARCHITECTURE.md`
High-level architecture doc. Target: 400-600 lines.

Cover:
- **Overview** — what flog is, the two-process model (TUI + Dart companion), terminal-native design choice
- **Four-layer architecture** — the `ui → app → domain ← parser/input` dependency arrow; why no cycles
- **Transport layer** — device discovery (flutter devices --machine polling), three connection methods (Localhost / AdbForward / Usbmuxd), port pool 19753+, reconnect backoff
- **Input/protocol layer** — WS protocol with Hello handshake, ClientMessage/ServerMessage wire types, session_id + connected_at metadata
- **Domain layer** — LogEntry/LogStore ring buffer (100K cap, fold-on-drain), NetworkEntry/NetworkStore (10K cap), FilterState/NetworkFilter (regex + plain OR-term merging), FlogNetKind typed enum + wire format
- **Parser layer** — MultiStrategyParser chain (structured → generic → keyword → network), shared ANSI stripping
- **App layer** — AppMode state machine, ViewTab, LogsViewState/NetworkState symmetry, ClickRegion enum + two-phase mouse dispatch
- **UI layer** — ratatui + Catppuccin Macchiato, dual-tab (Logs / Network), json_viewer + text_editor + input_field shared components
- **Concurrency model** — Tokio multi-threaded runtime, background device monitor/connector tasks, Arc<Mutex<App>>, main-thread event poll
- **Scroll model** — renderer is the scroll authority, auto_scroll per-view, Logcat-strict never-disturb semantics
- **Mock + Replay** — interceptor architecture, mock rule sync via ServerMessage, replay round-trip

Include ASCII diagrams where useful (layer cake, data flow). Cross-link to MODULES.md for per-file detail.
