# DART-033 — flog_dart v0.8 SSE architecture redesign

**Status:** shipped as `flog_dart 0.8.0` (2026-04-27).
**Plan:** [`docs/superpowers/plans/2026-04-27-dart-033-v08.md`](../plans/2026-04-27-dart-033-v08.md).
**Audit entry:** DART-033 — closed.

## Summary

flog_dart's SSE subsystem was a 417-line monolith (`flog_sse_parser.dart`)
that mixed four concerns: UTF-8 byte boundary handling, the W3C SSE line
parser, flog_net telemetry, and the [DONE]-filtering convenience. v0.8
splits it into three composable `StreamTransformer`s and keeps the old
`FlogSseParser` API as a thin compat shim.

```
bytes  → SseByteDecoder    → SseLineDecoder → FlogSseReporter → events
       (UTF-8 boundary,   (W3C parser as   (telemetry tee,
        BOM, bounded buf)  transformer)     tree-shakable)
```

Each layer is independently testable. `FlogSseReporter` is a compile-time
identity when `flogEnabled` is false, so AOT tree-shakes the entire
telemetry path in release builds.

## Migration surface

### Unchanged (source-compatible)

- `FlogLogger` — no touches.
- `FlogDio` non-SSE surface (get/post/put/etc., interceptors, config).
- `FlogWebSocket`, `FlogMockInterceptor`, `Flog.init`, `FlogServer`, `FlogStore`.
- `FlogSseParser.wrap` / `FlogSseParser.wrapTyped` — identical signatures,
  identical behavior, now delegate to the new transformers.
- Wire protocol (`t: req|chunk|done|err`, `p: sse`) — byte-identical.
- `SseEvent` — same shape, same equality, moved to `lib/src/sse/event.dart`
  and re-exported from `flog_sse_parser.dart` for the v0.7 import path.

### Breaking

- `SseResponse` constructor gains two required parameters: `events` and
  `options`. External code does not construct `SseResponse` directly —
  it's only returned by `FlogDio.sse()` / `flogSse()` — so this affects
  no public callers. Marked "breaking" for the semver audit trail only.

### Deprecated (still functional, removed in v1.0)

- `SseResponse.stream` — annotated `@Deprecated('Use .events for typed
  access; removed in v1.0')`. Continues to work exactly as in v0.7.

### New public API

Exported from `package:flog_dart/flog_dart.dart`:

- `SseByteDecoder` — `StreamTransformer<List<int>, String>`.
- `SseLineDecoder` — `StreamTransformer<String, SseEvent>`.
- `FlogSseReporter` — `StreamTransformer<SseEvent, SseEvent>`.
- `SseEvent` (already exported via the `flog_sse_parser.dart` re-export;
  now also directly exported for forward-compat).
- `SseResponse.events: Stream<SseEvent>`.
- `SseResponse.options: RequestOptions`.

## DART-033 audit-issue resolution

The 7 SSE-design issues the audit locked under DART-033:

1. **Layering mix** — byte decode + line parse + telemetry + [DONE]
   filter all in one class → now 3 named transformers + a thin shim.
2. **API asymmetry** — `wrap` returns `String`, `wrapTyped` returns
   `SseEvent`; no middle-ground for "typed without flog" → users can
   now build any pipeline they like by composing the transformers.
3. **Closure-variable state** — `_SseLineParser` held state in closure
   vars inside `_run` → the transformer owns a per-subscription
   `_LineDecoderState` object.
4. **UTF-8 decode cost** — `byteBuffer.sublist(...)` copied on every
   boundary split → now uses `BytesBuilder(copy: false)` and
   `Uint8List.sublistView` for zero-copy tail retention.
5. **Unbounded byte buffer** — OOM risk on a runaway SSE stream with no
   line boundaries → `SseByteDecoder(maxBufferBytes: 1 MiB)` with a
   descriptive `StateError` on overrun.
6. **Duplicate parser paths** — `wrap` and `wrapTyped` each ran the
   line parser separately (pre-Phase-3) → Phase 3 collapsed into `_run`;
   v0.8 collapses further into the shared transformer pipeline.
7. **Unexported `SseEvent` leak** — the type had to be imported from an
   internal path → now a first-class export of `flog_dart.dart`.

## Deviations from the plan

1. **Task 3 reporter frame key: `'done'` (not `'res'`).** The plan text
   says "emit 'res' flog_net event (completion summary)"; the plan's red
   line says "Behavior MUST match 0.7.x". v0.7 emits `'done'`; the flog
   Rust side parses `FlogNetKind::Done`. Using `'done'` preserves wire
   compat at the cost of a cosmetic divergence from the plan prose —
   preserving behavior beats preserving the colloquial name. Logged in
   the Task 3 commit message.
2. **Byte-level `size` dropped from `done` frame.** v0.7 emitted
   `{duration, chunks, size}`. The v0.8 `FlogSseReporter` is a
   `Stream<SseEvent>` transformer with no visibility into raw bytes, so
   `size` is omitted. No existing test or Rust consumer asserts on it;
   a future refinement could re-introduce it via a tap in the byte
   layer. Logged in the Task 3 commit.
3. **+33 tests, plan minimum was +28.** Extras landed where natural
   (byte decoder pause/resume, line decoder first-space-after-colon,
   SseResponse.options, null-body round-trip). No cut tests.

## Close-out

DART-033 is resolved. The Dart package is ready to publish as v0.8.0
(`flutter pub publish` requires interactive auth; the dry-run gate is
clean). The flog TUI does not need any changes — wire protocol is
unchanged, and the TUI decodes both v0.7.x and v0.8.x Dart clients
identically.
